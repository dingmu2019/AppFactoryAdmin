# API 管理功能作用分析报告

## 1. 概述
本系统中的“API管理”体系分为两个核心模块，分别服务于**接口查询**和**应用管控**两个不同的场景。

## 2. 接口文档中心 (API Definitions)
*   **页面位置**：系统管理 -> API 管理 (`/sys/api`)
*   **对应代码**：`frontend/src/pages/sys/api/page.tsx`
*   **核心作用**：作为系统对外开放接口的**在线目录和使用指南**。

### 主要功能
1.  **接口检索**：提供完整的 API 列表，支持按路径、请求方法（GET/POST）、业务分类进行搜索和过滤。
2.  **开发文档**：点击接口可查看详细的参数说明、鉴权要求（是否需要 Token/Key）。
3.  **测试辅助**：自动生成 `curl` 调用示例代码，方便开发者直接复制测试。
4.  **状态概览**：直观展示公开接口 (Public) 与受保护接口 (Protected) 的数量统计。

### 适用场景
供内部开发人员或第三方合作伙伴查询有哪些接口可用，以及如何调用这些接口。

---

## 3. 应用接入与鉴权 (Open API Platform)
*   **页面位置**：应用管理 -> OAuth Apps (`/apps`)
*   **对应代码**：`frontend/src/pages/apps/page.tsx` 及后端鉴权中间件
*   **核心作用**：管理调用 API 的**客户端应用及其权限**（即“谁可以调用 API”）。

### 主要功能
1.  **凭证管理**：创建和管理 App ID (API Key) 和 App Secret，支持密钥轮换（Rotate）以保障安全。
2.  **安全网关**：
    *   **IP 白名单**：限制只能从特定 IP 地址发起调用。
    *   **签名校验**：防止请求在传输过程中被篡改或重放。
3.  **能力配置**：为每个应用配置可使用的 AI 模型（如 GPT-4, Gemini）和其他第三方集成参数。
4.  **审计监控**：记录所有 API 的调用日志（`api_logs`），包括响应时间、状态码和来源 IP，用于排查问题和安全审计。

### 适用场景
当您需要将系统能力（如 AI 对话、数据查询）开放给其他系统（如小程序、企业微信、合作伙伴 ERP）调用时，在此处创建应用并分发密钥。

---

## 4. 总结
*   **左手查阅 (API 管理)**：看有什么接口，怎么调。
*   **右手管控 (应用管理)**：管谁在调，给不给调，调得对不对。

两者配合构成了完整的 API 治理体系：**API 管理页面定义了资产，应用管理页面控制了访问。**
