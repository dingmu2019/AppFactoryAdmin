# 全面系统审计与修复 Spec

## Why
为了确保系统在迁移和多次功能更新后，所有功能模块均能正常运行，且多语言（i18n）覆盖完整，无硬编码文案残留。这对于提升用户体验和系统稳定性至关重要。

## What Changes
- **功能审计**：全面扫描系统所有路由页面，验证核心交互（增删改查、特殊工具等）是否正常。
- **多语言检查**：检查所有页面是否存在未翻译的硬编码字符串，并补齐 `src/locales/index.ts` 中的缺失项。
- **缺陷修复**：针对审计中发现的运行时错误、API 失败或 UI 显示异常进行即时修复。
- **一致性验证**：确保中英文切换后，所有页面元素（标题、按钮、占位符、提示信息等）都能正确对齐。

## Impact
- **受影响范围**：整个 `src/app` 目录下的所有页面和 API 路由。
- **代码变动**：涉及多个组件文件的文案替换，以及 `src/locales` 字典文件的增补。

## ADDED Requirements
### Requirement: 自动化与人工结合审计
系统应通过自动化脚本扫描和人工页面遍历，识别所有潜在的功能失效点。

#### Scenario: 页面加载验证
- **WHEN** 用户导航至任意系统菜单
- **THEN** 页面应成功渲染，无 404 或运行时崩溃错误。

#### Scenario: 多语言完整性验证
- **WHEN** 切换语言至 EN 或 CN
- **THEN** 页面上不应出现另一种语言的硬编码文字。

## MODIFIED Requirements
### Requirement: 统一鉴权与日志
确保所有 `/api/database/*` 等敏感接口均已接入统一的管理员鉴权逻辑和审计日志记录（已在前期部分实现，需全局确认）。
